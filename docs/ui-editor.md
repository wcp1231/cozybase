# UI Editor

Cozybase 的可视化 UI 编辑器目前只在 Draft 模式提供，目标是让用户在 Agent 生成页面后做轻量微调，而不是替代 schema 或 Agent 工作流。

## 当前行为

- 仅 Draft 模式显示“编辑 UI”入口，Stable 保持只读。
- 编辑模式中的所有改动都先写入前端内存里的 `draftJson`，点击“保存”后才整体覆盖 `ui/pages.json`。
- 如果编辑期间后台或 Agent 改写了 `pages.json`，保存前会显示冲突提示；继续保存会覆盖外部改动。
- 复杂属性（对象、数组、动作配置、列配置等）统一通过 JSON 编辑器修改。

## 组件结构编辑

- 左侧面板包含页面列表、组件插入面板和组件树。
- 插入规则：
  - 选中可直接承载子节点的容器时，新增组件追加到该容器内部。
  - 选中普通节点时，新增组件插入到该节点之后。
  - 选中必填单节点槽位中的节点时，新增组件回退为插入到其父组件之后。
  - 没有选区时，新增组件追加到当前页面 `body` 末尾。
- 删除通过组件树触发，会删除目标节点及其整棵子树。
- 位于必填单节点槽位中的节点目前不会提供删除动作。

## 排序限制

- 组件树拖拽基于 `@dnd-kit/react` 和 `@dnd-kit/react/sortable`。
- 当前只支持同一父容器内的同层重排。
- 跨容器拖拽会被拒绝，原有层级关系保持不变。
